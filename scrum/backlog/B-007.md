## Objective

```yaml

title: Objective
status: active
owner: Guilherme
tags: [objective]
```

Implement `smd init` so a project can be initialized from a signed template package produced by `smd pack`.

## Scope

```yaml
section: backlog.item.B-007.scope
title: Scope
status: active
owner: Guilherme
in_scope:
  - Add a public `smd init` command.
  - Accept a local signed template package as input.
  - Verify or enforce the package declaration and signing policy established by B-006.
  - Collect or accept the initialization inputs required by the specification.
  - Render the initial Scrum memory files into the target project root.
  - Create `.smd/config.yml` with safe defaults and selected options.
  - Refuse to overwrite existing memory unless explicitly allowed.
  - Return created files, selected template identity, warnings, and errors through the stable JSON envelope.
  - Add automated CLI tests for successful initialization, invalid package, existing memory protection, and required input validation.
out_of_scope:
  - Web download of template packages.
  - Interactive TUI or wizard beyond command-line options.
  - Remote trust registry or key server integration.
  - Full backlog, sprint, or gate workflow commands.
tags: [scope, cli, init, templates, signature]
```

## Acceptance Criteria

```yaml
section: backlog.item.B-007.acceptance-criteria
title: Acceptance Criteria
status: active
owner: Guilherme
criteria:
  - `smd init --template-package <file.zip> --json` initializes Scrum memory in a target root.
  - The command depends on and respects the manifest and signing policy defined in B-006.
  - The command creates the required base memory files from the template package.
  - The command writes `.smd/config.yml` with selected initialization options.
  - Existing Scrum memory is protected from accidental overwrite.
  - Invalid package, missing required inputs, and overwrite conflicts return structured errors.
  - Automated tests cover success and failure paths.
tags: [acceptance, cli, init, tests]
```
