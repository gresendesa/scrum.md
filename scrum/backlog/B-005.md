## Objective

```yaml
section: backlog.item.B-005.objective
title: Objective
status: active
owner: Guilherme
tags: [objective]
```

Create a concise, scenario-driven manual of best practices and practical usage patterns for LLM agents operating Scrum memory through `smd`, packaged with each template `.zip`.

## Scope

```yaml
section: backlog.item.B-005.scope
title: Scope
status: active
owner: Guilherme
in_scope:
  - Document high-value `smd` command patterns for session start, backlog refinement, sprint planning, sprint start, execution, review, closure, and retrospective.
  - Make the manual part of each template package so every `.zip` ships with its own LLM instructions.
  - Organize guidance around real scenarios an LLM will face while acting as Scrum Master and Developer.
  - Cover Scrum ritual flow, state transitions, transition gates, and what the agent must verify before moving records.
  - Explain how agents should query open backlog, active sprint, context, impact, validation, and gates through `smd`.
  - Include information-seeking patterns for finding relevant backlog, sprint, decisions, experiences, architecture notes, dependencies, and impact.
  - Include prompt-level habits and command sequences that prevent state drift, duplicate IDs, false `done`, skipped rituals, and direct Markdown guessing.
  - Clarify that LLM agents should use `smd`; MDBind is an internal implementation detail encapsulated by `smd`.
out_of_scope:
  - Teaching agents to call `mdb` directly in normal workflows.
  - Replacing the formal technical specification.
  - Building new CLI commands.
  - Remote template download mechanics.
tags: [scope, documentation, llm, smd]
```

## Acceptance Criteria

```yaml
section: backlog.item.B-005.acceptance-criteria
title: Acceptance Criteria
status: active
owner: Guilherme
criteria:
  - Manual is concise and optimized for realistic LLM operational scenarios.
  - Manual is included in the template `.zip` package layout.
  - Manual can be specific to the template package instead of being one global generic document only.
  - Manual lists recommended `smd` command sequences by Scrum ritual and workflow phase.
  - Manual explains how agents should handle state transitions and gates before changing backlog or sprint status.
  - Manual shows how to search for relevant project memory before planning or implementation.
  - Manual explicitly says agents should not use `mdb` directly during normal operation.
  - Manual explains that `smd` encapsulates MDBind graph operations.
  - Manual includes examples for discovering open backlog without leaking implementation details.
tags: [acceptance, documentation]
```
