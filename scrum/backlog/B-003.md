Create the memory foundation that lets an agent inspect Scrum state deterministically without manual Markdown guessing.

## Scope

```yaml
section: backlog.item.B-003.scope
title: Scope
status: active
owner: Guilherme
in_scope:
  - Resolve memory root from config and caller input.
  - Discover backlog, sprint, decision, and experience records.
  - Find active sprints and pending backlog items from structured metadata.
  - Calculate next backlog, sprint, decision, and experience IDs.
  - Integrate MDBind as a Python library for graph validation.
  - Expose reusable memory/query primitives for later CLI commands.
out_of_scope:
  - Typer command routing and user-facing CLI shell.
  - Full backlog and sprint file mutation commands.
  - Git audit enforcement.
tags: [scope, memory, mdbind, validation]
```

## Acceptance Criteria

```yaml
section: backlog.item.B-003.acceptance-criteria
title: Acceptance Criteria
status: active
owner: Guilherme
criteria:
  - The repository can locate required Scrum memory files from a configurable root.
  - Active sprint and pending backlog queries work without text parsing.
  - ID helpers calculate the next valid IDs without reusing existing IDs.
  - MDBind validation is invoked through Python library APIs rather than shelling out to `mdb`.
  - Validation results can be converted into structured errors and warnings.
  - Automated tests cover clean memory, duplicate IDs, missing files, and MDBind validation failure cases.
tags: [acceptance, memory, mdbind, tests]
```
