## Objective

```yaml

title: Objective
status: active
owner: Guilherme
tags: [objective]
```

Create the executable Python CLI shell for `smd`, exposing the memory foundation through Typer commands and a stable JSON contract.

## Scope

```yaml
section: backlog.item.B-004.scope
title: Scope
status: active
owner: Guilherme
in_scope:
  - Add a Python package and `smd` entrypoint using Typer.
  - Implement global options `--json`, `--root`, `--quiet`, and `--verbose`.
  - Load `.smd/config.yml` with safe defaults.
  - Implement the JSON output envelope defined by the specification.
  - Add thin command handlers for foundation queries, backed by B-003 primitives.
  - Add a thin template command that loads a local template `.zip` package and invokes B-002 rendering primitives to create files.
  - Define the initial template package structure, including Jinja files, package metadata, and LLM instruction/manual files.
out_of_scope:
  - Full backlog, sprint, gate, and context command behavior.
  - Template rendering internals beyond invoking B-002 primitives.
  - Full `smd init` project bootstrap workflow.
  - Web download of remote template packages.
  - Template trust, signature, checksum, cache, and version policy.
  - Git audit and protected-file diff enforcement.
tags: [scope, cli, json, config]
```

## Acceptance Criteria

```yaml
section: backlog.item.B-004.acceptance-criteria
title: Acceptance Criteria
status: active
owner: Guilherme
criteria:
  - `smd --help` exposes the CLI entrypoint and documented global options.
  - Foundation commands can emit the stable JSON envelope with `ok`, `command`, `timestamp`, `root`, `data`, `warnings`, and `errors`.
  - A public `smd` command can render Scrum memory from a local `.zip` template package into a target root.
  - Template-render command reports created files through the JSON envelope.
  - Template package includes Jinja templates and LLM-facing instruction files.
  - Invalid or incomplete template packages fail with structured errors.
  - Errors return non-zero exit codes and structured error objects.
  - Configuration is loaded from `.smd/config.yml` when present.
  - Automated tests cover CLI invocation and JSON envelope determinism.
tags: [acceptance, cli, json, tests]
```
