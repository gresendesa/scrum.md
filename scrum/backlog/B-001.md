## Acceptance Criteria

```yaml
section: backlog.item.B-001.acceptance-criteria
title: Acceptance Criteria
status: active
owner: Guilherme
criteria:
  - smd init creates the required scrum memory files from templates.
  - smd backlog create calculates the next B-XXX ID and creates synchronized consolidator/detail records.
  - smd backlog list can list pending or status-filtered items and supports JSON output.
  - smd sprint create calculates SPR-YYYY-NN, creates a detail file, and prevents accidental parallel active sprints.
  - smd sprint active reports the active sprint or null in the standard JSON envelope.
  - smd validate detects missing required files, duplicate sections, duplicate IDs, broken references, invalid statuses, and consolidator/detail mismatches.
  - All implemented commands accept --json, --root, --quiet, and --verbose.
  - Automated tests cover the implemented command behavior.
tags: [acceptance, quality]
```
