## Objective

```yaml

status: active

```

Create Jinja2 templates that render valid Scrum memory optimized for agent operation, graph validation, state queries, and context retrieval.

## Scope

```yaml
section: backlog.item.B-002.scope
title: Scope
status: active
owner: Guilherme
in_scope:
  - Define the runtime template directory layout for the Python package.
  - Convert the current default memory model into Jinja2 templates.
  - Implement template loading and rendering primitives.
  - Support global variables from `.smd/config.yml` and command input.
  - Ensure rendered memory supports `mdb validate`, status search, backlinks, and context lookup.
  - Provide test fixtures for rendering templates into a temporary memory root.
out_of_scope:
  - Full `smd init` command behavior.
  - Backlog and sprint mutation commands.
  - Interactive template selection UI.
tags: [scope, templates, jinja2]
```

## Acceptance Criteria

```yaml
section: backlog.item.B-002.acceptance-criteria
title: Acceptance Criteria
status: active
owner: Guilherme
criteria:
  - Jinja2 templates exist for the required base memory files and detail records.
  - Template rendering can create deterministic Markdown output in a temporary directory.
  - Rendered Markdown includes valid MDBind `section` metadata.
  - Rendered memory passes `mdb validate --root . --json`.
  - Rendered records can be found with `mdb search status=...` without ad hoc text parsing.
  - Rendered sprint and backlog records can be inspected with `mdb context`.
  - Template variables are explicit, documented in tests, and sourced from config or command input.
  - Automated tests cover successful rendering and missing required variable handling.
tags: [acceptance, templates, tests]
```
